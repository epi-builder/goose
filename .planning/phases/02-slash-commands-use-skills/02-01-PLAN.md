## Trust Boundaries
- Shared core discovery logic ↔ CLI/server surfaces that present slash suggestions
- Server slash-command contract ↔ desktop generated clients consuming canonical skill reference metadata

## STRIDE Register
| threat_id | category | component | disposition | mitigation_plan |
| --- | --- | --- | --- | --- |
| T-02-01 | Tampering | Skill discovery consistency | mitigate | Reuse `summon` discovery logic directly or extract a shared helper from it; do not create a parallel slash-only registry. |
| T-02-02 | Information Disclosure | Canonical skill reference path payload | mitigate | Only expose resolved, already-allowed skill source paths from `summon` results; do not add arbitrary filesystem probing in server or UI. |
| T-02-03 | Repudiation | Cross-surface traceability | mitigate | Keep slash-command types explicit and document generated API contract changes so contributors can trace skill metadata from server to desktop. |
| T-02-04 | Elevation of Privilege | Slash completion vs actual executable skill set | mitigate | Ensure listed skills and executable skills come from the same source set for the same working directory to prevent suggesting inaccessible skills. |
</threat_model>
